# Legal Document Generator

## When to Use

When a project needs customer-facing legal documents drafted from templates.
Every output document carries a "DRAFT — LAWYER REVIEW REQUIRED" header.
The framework can produce starting points; a Singapore lawyer must review
before any binding use.

## Procedure

## Step 1 — Determine Scope

From `$ARGUMENTS`:
- `privacy` — Privacy Policy only
- `tos` — Terms of Service only
- `dpa` — Data Processing Agreement only
- `cookie` — Cookie Policy only
- `all` — generate all four
- If empty: ask which documents are needed

## Step 2 — Read Data Inventory

Read `wiki/compliance/data-inventory.md` if it exists.
The Privacy Policy and DPA need to accurately reflect what PII is processed.
If data-inventory.md doesn't exist: STOP and suggest running `/data-inventory` first.

## Step 3 — Collect Project Identity

Read project CLAUDE.md and ask for any missing details:
- Legal entity name (e.g., "Acme Pte Ltd")
- Business address (Singapore-based for PDPA)
- DPO name and contact (can be founder for early-stage)
- Jurisdiction (PDPA-Singapore by default)
- End-user jurisdiction (Singapore only? Global?)
- Product name and one-line description

## Step 4 — Generate Documents

For each document requested, copy the template from
`~/.claude/templates/legal/{document}-template.md` to
`wiki/legal/{document}.md` and substitute placeholders.

Templates use placeholder variables:
- `{{LEGAL_ENTITY}}` — legal entity name
- `{{ADDRESS}}` — business address
- `{{DPO_NAME}}` — DPO name
- `{{DPO_EMAIL}}` — DPO email
- `{{PRODUCT_NAME}}` — product name
- `{{PII_FIELDS_LIST}}` — from data inventory
- `{{EXTERNAL_PROCESSORS}}` — from data inventory (vendors that see PII)
- `{{RETENTION_TABLE}}` — from data inventory retention column
- `{{CROSS_BORDER_TRANSFERS}}` — from data inventory transfers
- `{{EFFECTIVE_DATE}}` — today's date
- `{{LAST_UPDATED}}` — today's date

Each output file MUST start with this header:

```markdown
> ⚠️ **DRAFT — LAWYER REVIEW REQUIRED**
>
> This document was generated by the Claude Code Framework as a starting point.
> Singapore PDPA compliance requires a lawyer's review before any binding use.
> Do NOT sign, publish, or share with clients until a qualified lawyer has
> reviewed and adapted this document to your specific circumstances.
>
> Generated: {YYYY-MM-DD}
> Project: {project name}
> Generator: legal-docs skill v1.0
```

## Step 5 — Check Data Inventory Alignment

Verify generated documents are consistent with the actual data inventory:
- Privacy Policy mentions every PII field collected
- DPA mentions every external processor
- Retention sections match data-inventory.md
- Cross-border transfers section matches data-inventory.md

If misalignment found, fix it in the generated doc, not by modifying the inventory.

## Step 6 — Report

```
Legal documents generated

  Privacy Policy:  wiki/legal/privacy-policy.md   [DRAFT — REVIEW]
  Terms of Service: wiki/legal/terms-of-service.md [DRAFT — REVIEW]
  DPA:             wiki/legal/dpa.md              [DRAFT — REVIEW]
  Cookie Policy:   wiki/legal/cookie-policy.md    [DRAFT — REVIEW]

⚠ NONE of these are ready for use. Required next steps:
  1. Read each document end-to-end and adjust to match your actual practices
  2. Engage a Singapore lawyer for review (typical cost: S$1,500–3,000 for the set)
  3. Once approved by lawyer, publish on your website / share with clients
  4. Set a 12-month review reminder — privacy laws evolve

Suggested lawyer specializations:
  - Singapore PDPA practitioner
  - Tech/SaaS contracts background
```

## Pitfalls

- Treating generated documents as ready-to-use — they aren't. Lawyer review is non-negotiable.
- Generating a Privacy Policy that doesn't match the actual data inventory — credibility-killer in audits
- Forgetting to update the docs when data inventory changes — they drift apart
- Using generic templates without adapting to actual operations — lawyer will reject
- Generating without data-inventory.md existing — you'll list PII you don't actually collect (or miss what you do)

## Verification

- Each requested document was written to `wiki/legal/`
- Each document has the "DRAFT — LAWYER REVIEW REQUIRED" header at the top
- Each document references actual PII fields from data-inventory.md
- External processors match data-inventory.md
- User has been told explicitly that lawyer review is required
